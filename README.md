## Features

### Authentication
This project uses third provider services to manage some information and one of them is user related data. So, our authentication is done by using a JWT token generated by other systems. To handle it, we have:
- `AuthGuard` is a nest Guard that handles this token in every request, basically it checks if it's a private route. If so, it decodes the third party token. It's already setup for it, so you might not interact with its code;
- `@Authorized` is a class and method decorator that turns one or more routes into private ones. See its file (src/common/auth/Authorized.ts) to understand how to use it.
- `@CurrentUser` is parameter decorator that can inject a variable named `user` into a controller method. See its file (src/common/auth/CurrentUser.ts) to understand how to use it.

**How to generate a valid token?**

Now, we are using tokens provided directly by Smiles systems. To generate a token:
1. Access the main web app (smiles.hm.outsmartdigital.com.br)
2. Sign In (if you do not have an user, ask someone from your team)
3. Access page that returns the token itself (smiles.hm.outsmartdigital.com.br/api/auth/accessToken)


**How to send token**
Send a JWT token through `Authorization` header

### Controllers
Every controller must return an object according to following contract:
```js
{
  entity: {}, // response body
  statusCode: 0, // http status code
  message: '' // responde message
}
```
